# BuilderHub Community Learning Roadmap

**Goal:** Take beginners from zero/low confidence → able to contribute to a real fullstack app.

Each phase includes:

* Concepts to learn
* Practical exercises
* Direct project contributions
* Deliverables

---

# Phase 0 — Foundations & Environment (Week 1)

Most communities skip this and regret it. Don’t. Tooling confidence reduces dropout.

## Learn

* How the web works (client/server basics)
* What frontend vs backend means
* Git basics
* GitHub workflow
* Terminal basics
* Code editor setup

## Practice

* Install Node / Bash / Git
* Create GitHub account
* Clone repo
* Make first commit
* Push branch
* Open first PR (even typo fix)

## Project Contribution

* Fix README typo
* Add your name to CONTRIBUTORS.md
* Update small docs

## Deliverable

- First merged PR

Psychological win > technical complexity here.

---

# Phase 1 — Frontend Core (Weeks 2–3)

We start frontend first because visual feedback keeps beginners motivated.

## Learn

* HTML structure
* CSS basics
* Layout & responsiveness
* JavaScript fundamentals
* React basics
* Components
* Props
* State
* Forms

## Practice

* Build static pages
* Create reusable components
* Form validation
* UI layouts

## Project Contribution

Frontend team builds:

* Landing page
* Navigation bar
* Footer
* Signup form UI
* Login form UI
* Resource list page

## Deliverable

- Working frontend screens (no backend yet — mock data allowed)

---

# Phase 2 — Backend Fundamentals (Weeks 4–5)

Now we give the frontend something real to talk to.

## Learn

* What APIs are
* REST basics
* Express or Django basics
* Routing
* Controllers
* JSON responses
* CRUD operations
* Middleware

## Practice

* Build simple API
* Create endpoints
* Return mock data
* Handle requests

## Project Contribution

Backend team builds:

* User routes
* Resource routes
* Challenge routes
* API structure
* Error handling

## Deliverable

- API endpoints working locally

---

# Phase 3 — Database & Auth (Weeks 6–7)

This is where devs start feeling “real”.

## Learn

* Databases (SQL vs NoSQL overview)
* Schema design basics
* Models
* Relationships
* Authentication concepts
* Hashing passwords
* JWT vs sessions

## Practice

* Create DB models
* Save user records
* Query data
* Login flow

## Project Contribution

Add:

* User model
* Registration endpoint
* Login endpoint
* Protected routes
* Profile data storage

## Deliverable

- Users can register and login

---

# Phase 4 — Frontend + Backend Integration (Weeks 8–9)

Now it becomes a real app.

## Learn

* Fetch / Axios
* API consumption
* Loading states
* Error handling UI
* Token handling

## Practice

* Connect forms to backend
* Handle API errors
* Display real data

## Project Contribution

Connect:

* Signup form → backend
* Login form → backend
* Profile page → API
* Resource list → API

## Deliverable

- End-to-end working flow

---

# Phase 5 — Collaboration & Code Quality (Week 10)

Teach them how real teams work.

## Learn

* Git branching strategy
* PR reviews
* Commit message standards
* Code readability
* Basic testing
* Linting

## Practice

* Review each other’s PRs
* Refactor small modules
* Add tests

## Project Contribution

* Code cleanup
* Add validation
* Add tests
* Improve docs

## Deliverable

- First reviewed & improved features

---

# Phase 6 — Feature Sprint (Weeks 11–12)

Now let teams build independently.

## Feature Options

Teams choose one:

* Progress tracker
* Coding challenges module
* Badge system
* Study group module
* Discussion board

## Learn

* Feature design
* Task breakdown
* Sprint workflow
* Demo presentation

## Deliverable

- Feature demo day

---

# Weekly Session Structure

Each week:

**Session A — Learning**

* Teach concepts
* Walkthrough demos

**Session B — Build & Review**

* Hands-on coding
* Pair programming
* PR reviews
* Bug fixing
* Show progress
